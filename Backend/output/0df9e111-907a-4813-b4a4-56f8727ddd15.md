# File Concepts and Access Methods
=====================================

## Introduction

An operating system provides a uniform logical view of data storage by abstracting physical storage into a logical unit called a file. Files are mapped onto physical storage devices (secondary memory) and serve as the smallest unit of logical secondary storage.

## File Attributes
-----------------

A file is named for convenience, referred to by its name, and becomes independent of the process, user, and system that created it. The attributes of a file may vary depending on the operating system but typically include:

*   Name
*   Identifier
*   Type
*   Location
*   Size
*   Protection
*   Time, date, and user identification

## File Operations
-----------------

The operating system performs various basic file operations:

*   Creating a File
*   Writing to a File
*   Reading from a File
*   Repositioning within a File
*   Deleting a File
*   Truncating a File

## File System Structure
----------------------

A file system is the organization of files on secondary storage (disks). It provides a user interface to storage, mapping logical to physical. The structure consists of:

*   A file control block that stores information about a file
*   Device drivers that manage I/O devices at the I/O control layer
*   File systems organized into layers

### File System Layers
---------------------

#### **I/O Control Layer**
*   Manages I/O devices and translates high-level commands to low-level hardware-specific commands.

#### **File Organization Module**
*   Translates logical block numbers to physical block numbers and manages free space, disk allocation.

#### **Logical File System**
*   Manages metadata information and translates file names into file numbers.
*   Directory management
*   Protection

## Access Methods
-----------------

An operating system uses different access methods to manage files:

*   **Sequential Access:** In this method, the operating system accesses files in a sequential manner, one after another.

*   **Direct Access:** This method allows direct access to any location in a file without traversing the entire file.

*   **Indexed Access:** Indexed access is a method of accessing files by using an index or table that maps logical block numbers to physical block numbers. It provides faster access times compared to sequential or direct methods.

## Key Takeaways
----------------

-   Files are abstracted logical units that map onto physical storage devices.
-   File attributes include name, identifier, type, location, size, protection, and time, date, and user identification.
-   Operating systems perform basic file operations such as creating, writing to, reading from, repositioning within, deleting, and truncating files.
-   A file system provides a structure for organizing files on secondary storage.
-   File systems are organized into layers that manage I/O devices, allocate space, manage metadata, and provide directory management and protection.

## File System Layers
### Overview

File systems are crucial components of operating systems, managing metadata and providing data storage for applications. The layering of file systems allows for reduced complexity and redundancy but introduces overhead and potential performance degradation.

### Definitions

- **Logical file system**: A higher-level abstraction that translates file names into file numbers, handles, and locations using file control blocks (inodes in Unix).
- **Layering**: A design approach that reduces complexity by decomposing the file system into multiple layers, each with its own characteristics and advantages.

### Key Concepts

* Multiple logical file systems often coexist within an operating system.
* Each file system has its unique format, such as ISO 9660 for CD-ROMs or NTFS for Windows.

### Layering Advantages and Disadvantages

- **Advantages**: Reduces complexity, increases flexibility, and allows for easier maintenance and upgrades.
- **Disadvantages**: Introduces overhead and potential performance degradation due to the additional layer of abstraction.

### File System Structures in Memory

* **Mount table**: Stores file system mounts, mount points, and file system types.
* **Buffers**: Hold data blocks from secondary storage, facilitating efficient data access.

### Access Methods

#### Overview

Files store information that must be accessed and read into computer memory. The information can be accessed through various methods: sequential, direct, and index access.

#### Sequential Access

- **Definition**: The simplest access method where information is processed in order, one record after the other.
- **Characteristics**:
	+ Reads and writes make up the bulk of operations on a file.
	+ A read operation advances a file pointer, while a write operation appends to the end of the file.

#### Direct Access

- **Definition**: Also known as relative access, where files are viewed as numbered sequences of blocks or records for rapid reading and writing in any order.
- **Characteristics**:
	+ The direct-access method is based on a disk model of a file.
	+ There are no restrictions on the order of reading or writing.

#### Index Access

- **Definition**: An access method that uses an index to find a record in the file, first searching the index and then using the pointer to access the desired record.
- **Characteristics**:
	+ The index contains pointers to various blocks.
	+ To find a record, we search the index and then use the pointer to access the file directly.

### Key Takeaways

* Files are logical units of storage used to organize and manage data in an operating system.
* Different access methods (sequential, direct, indexed) determine how data is read and written efficiently.
* The layering approach reduces complexity but introduces overhead and potential performance degradation.

# Directory Structure and Operating Systems
=============================================

## Introduction

An operating system (OS) is a software that manages computer hardware resources and provides common services to computer programs running on the hardware. The OS acts as an intermediary between the user and the computer hardware.

### Definition of Operating System

An operating system is a software in a computer system that manages the system's hardware and software resources, providing a platform for software applications to run on.

## Directory Structure

A file is defined by its metadata, which includes:

*   Operating system: The OS stores information about files in various data structures such as file systems and directories.
*   Compiler: Although compilers do not directly store file metadata, they are involved in the process of creating executable files from source code. However, this does not determine the fundamental structure of a file.
*   User/creator: Users or creators typically manage and organize their own files within the directory structure.
*   Hardware: The physical hardware components of a system do not define the information contained in a file.

### Indexed Access Method

Indexed access methods use:

*   Index table: An index table is used to quickly locate specific data within a database or file system. It serves as a map between key values and their corresponding locations, enabling efficient searching.
*   Stack structure: Stack structures are not typically used for indexed access in file systems or databases. Instead, they are often employed in programming languages to implement recursive functions and handle function calls.
*   Queue structure: Queue data structures are also not commonly used for indexed access. They are more frequently utilized in job scheduling, print queues, and network communication.

## Key Takeaways

*   An operating system is a software that manages computer hardware resources and provides common services to computer programs running on the hardware.
*   The directory structure of a file system is defined by metadata such as the user/creator who manages it, rather than by the operating system itself or compiler directly.
